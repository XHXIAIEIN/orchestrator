# Data Contract — User/System Layer Separation

> **Purpose**: Define which files are user data (never auto-updated) vs system data (safe to replace).
> Any automation (compiler, updater, hooks) MUST consult this contract before modifying files.

## Layers

### User Layer (NEVER auto-modify)

These files contain user customizations, personal data, or accumulated knowledge.
Automation may READ them but must NEVER overwrite, delete, or merge into them without explicit user consent.

| Path | Content | Why protected |
|------|---------|---------------|
| `SOUL/private/identity.md` | AI identity / personality core | Personal definition |
| `SOUL/private/relationship.md` | User-AI relationship context | Personal |
| `SOUL/private/hall-of-instances.md` | Instance history & memories | Accumulated knowledge |
| `SOUL/private/experiences.jsonl` | Behavioral calibration data | Training data |
| `SOUL/private/calibration.jsonl` | Voice calibration samples | Training data |
| `SOUL/private/voice.md` | Voice style definition | Personal |
| `SOUL/private/session-state.md` | Cross-session state | Accumulated state |
| `SOUL/private/prompts/*.md` | Private prompt overrides | User customization |
| `.claude/hooks/*.sh` | Hook scripts | User-configured behavior |
| `.claude/hooks/*.conf` | Hook configurations | User-configured rules |
| `.claude/settings.json` | Claude Code settings | User preferences |
| `.env` | Environment variables | Secrets & config |
| `config/*.yml` | Runtime configuration | User-tuned parameters |
| `data/*.db` | SQLite databases | Accumulated data |
| `data/*.jsonl` | Event logs | Accumulated data |

### System Layer (safe to auto-update)

These files are system code and templates. They can be replaced by updates,
regenerated by the compiler, or refactored without losing user work.

| Path | Content | Update mechanism |
|------|---------|-----------------|
| `src/**/*.py` | Core engine code | git pull / refactor |
| `SOUL/public/prompts/*.md` | System prompt templates | compiler / git pull |
| `SOUL/public/references/*.md` | Reference documentation | git pull |
| `SOUL/public/steal/*.md` | Steal reports (read-only analysis) | steal skill |
| `.claude/skills/*/SKILL.md` | Skill definitions | skill updates |
| `.claude/skills/*/constraints/*.md` | Skill hard rules | skill updates |
| `.claude/agents/*.md` | Agent definitions | agent updates |
| `departments/*/blueprint.yaml` | Department configs | system refactor |
| `departments/*/SKILL.md` | Department prompts | system refactor |
| `docs/**/*.md` | Documentation | git pull |
| `bin/*.sh` | Utility scripts | git pull |
| `dashboard/**` | Web dashboard | git pull |
| `docker-compose.yml` | Container config | git pull (review first) |
| `Dockerfile` | Build definition | git pull |

### Hybrid Layer (auto-update with merge strategy)

These files have both system structure and user content. Updates must MERGE, not replace.

| Path | Content | Merge strategy |
|------|---------|---------------|
| `CLAUDE.md` | Project instructions | Append-only for new rules; never remove existing |
| `.claude/boot.md` | Compiled boot prompt | Regenerated by compiler from sources |
| `SOUL/tools/compiler.py` | Boot compiler | System code, but reads User Layer |
| `docs/architecture/PATTERNS.md` | Pattern ledger | Append new entries; never modify existing |

## Rules

1. **Read before write**: Any script that modifies files MUST check this contract first.
2. **User Layer is sacred**: Even `git pull` should not overwrite User Layer files (use `.gitignore`).
3. **System Layer is replaceable**: These files can be regenerated from source. No user state lives here.
4. **Hybrid requires merge**: Never `cp` over a Hybrid file. Diff, merge, or append.
5. **New files default to System**: Unless they contain user data, new files are System Layer.
6. **Promotion path**: A System file becomes User Layer when the user customizes it. Track in this contract.
7. **Compiler reads this**: `SOUL/tools/compiler.py` should import this contract to know what's safe to touch.

## Verification

To check contract compliance:
```bash
# List all User Layer files that have been modified in the last commit
git diff --name-only HEAD~1 | while read f; do
  grep -q "$f" DATA_CONTRACT.md && echo "CHECK: $f may be User Layer"
done
```
