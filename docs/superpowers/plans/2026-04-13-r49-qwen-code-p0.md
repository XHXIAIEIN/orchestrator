## Steps

### Step 1: BlockStreamer Python 移植 (~60 min)

**Create** `src/channels/block_streamer.py`

Port from `packages/channels/base/src/BlockStreamer.ts` (134 LOC → ~100 LOC Python):
- `push(chunk)` — 累积 buffer + 检查 emit 触发
- `check_emit()` — maxChars 强制切 + minChars 段落边界切
- `_on_idle()` — idleMs 超时自动 flush
- `flush()` — 发送所有剩余
- `_emit_block(text)` — 串行化发送（threading.Lock 或 asyncio.Lock）
- `_find_break_point()` / `_find_block_boundary()` — 段落 > 换行 > 空格 > 硬切

关键适配：
- TypeScript promise chain → Python `threading.Lock` + queue（sync 版本先行）
- 可选 async 版本用 `asyncio.Lock`
- 默认阈值: minChars=400, maxChars=1000, idleMs=1.5

→ verify: `python -c "from src.channels.block_streamer import BlockStreamer; print('ok')"`

### Step 3: CLIBridge 流式读取 (~90 min)

**Modify** `src/channels/agent_bridge.py`

`CLIBridge.chat()` 改为支持 streaming:

```python
async def chat_stream(self, message, system_prompt=""):
    """Streaming variant — yields text chunks as they arrive."""
    cmd = [self.profile.binary_path, "--output-format", "stream-json",
           "--print", message]
    proc = subprocess.Popen(cmd, stdout=PIPE, stderr=PIPE, text=True, cwd=self._cwd)

    for line in proc.stdout:
        event = json.loads(line.strip())
        if event.get("type") == "assistant" and event.get("subtype") == "text":
            yield event["content"]
        elif event.get("type") == "result":
            self._session_id = event.get("session_id", self._session_id)
```

保留同步 `chat()` 不变（向后兼容），新增 `chat_stream()` 作为可选路径。

depends on: none
→ verify: 手动测试 `claude --output-format stream-json -p "hello"` 确认输出格式

### Step 4: TG Channel 集成 (~60 min)

**Modify** `src/channels/telegram/channel.py` + `handler.py`

将 `_start_chat` 改为：
1. 根据 dispatch_mode 调用 `ConversationLockManager`
2. 如果启用了 block_streaming，创建 `BlockStreamer(send_fn=self._send_text)`
3. 调用 `do_chat` 时传入 `on_chunk` callback
4. `on_chunk` → `streamer.push(chunk)`
5. 完成后 `streamer.flush()`

**Modify** `src/channels/chat/engine.py`

`do_chat` 支持 `on_chunk` 可选参数：
- Claude API 的 streaming mode (`stream=True`) → yield 每个 text delta → `on_chunk(delta)`
- 本地模型路径暂不改（local 模型通常很快，不需要分段）

depends on: step 1, step 2, step 3
→ verify: 发 TG 消息，确认收到分段回复而非一整段

## Risk Notes

1. **stream-json 格式未标准化**: Claude Code 的 stream-json 输出格式可能随版本变化。加 try/except 容错 + 日志，graceful degradation 回退到非流式。
2. **steer 模式需要取消正在运行的 `do_chat`**: 当前 `do_chat` 是同步阻塞的，取消需要线程中断或标记检查。初版可用 `cancelled` flag + 每轮循环检查。
3. **BlockStreamer 线程安全**: TG handler 在独立线程中运行，BlockStreamer 的 idle timer 需要线程安全的 timer 实现。
