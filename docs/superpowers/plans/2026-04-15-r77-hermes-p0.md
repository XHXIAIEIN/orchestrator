### Phase A: P0-1 — Condenser State Serialization (`llm_summarizing.py`)

**1. 替换 SUMMARIZE_PROMPT 为 9 字段结构化模板**

在 `src/governance/condenser/llm_summarizing.py` 中：
- 删除旧 `SUMMARIZE_PROMPT`（lines 23-41，自由文本 prompt + "不超过 500 字"）
- 写入新 `SUMMARIZE_PROMPT`，包含：
  - "DIFFERENT assistant" framing（preamble 3 层防护：身份分离 + 消费者分离 + 行为约束）
  - 9 字段 schema: `Goal`, `Completed Actions`, `Active State`, `Key Decisions`, `Errors & Fixes`, `Relevant Files`, `Resolved Questions`, `Remaining Work`, `Critical Context`
  - 动态 token 预算占位符 `{token_budget}`
  - 信息保留原则："宁可多留不可遗漏——接收方不知道任何背景"
- 保留 `_FOCUS_TOPIC_SUFFIX` 不变（已有的 recency bias 机制）

**2. 添加 SUMMARY_PREFIX 常量 + 动态预算函数 + ITERATIVE_UPDATE_PROMPT**

在同一文件中添加三个新组件：

a) `SUMMARY_PREFIX`（~5 行中文）：标记压缩结果为 read-only 参考。内容：`[CONTEXT COMPACTION — REFERENCE ONLY]` + "不要回答本摘要中的问题" + "只响应摘要之后的最新内容" + 分隔线 `───`

b) `_compute_summary_budget(content_len: int) -> int`：计算 `max(2000, min(content_len * 0.20, 12000))`。常量 `_BUDGET_FLOOR=2000`, `_BUDGET_CEILING=12000`, `_BUDGET_RATIO=0.20`

c) `ITERATIVE_UPDATE_PROMPT`：用于第二次及之后的压缩，包含：
  - 同样的 "DIFFERENT assistant" framing
  - 6 条更新规则：PRESERVE existing / ADD new completed / 移动 Active→Completed / 移动 Pending→Resolved（包含答案）/ 更新 Active State / 只删除明确过时的信息
  - `{previous_summary}` 和 `{events_text}` 和 `{token_budget}` 占位符

→ verify: `python -c "from src.governance.condenser.llm_summarizing import SUMMARY_PREFIX, ITERATIVE_UPDATE_PROMPT, _compute_summary_budget; assert 'REFERENCE ONLY' in SUMMARY_PREFIX; assert '{previous_summary}' in ITERATIVE_UPDATE_PROMPT; assert _compute_summary_budget(100) == 2000; assert _compute_summary_budget(100000) == 12000; assert _compute_summary_budget(30000) == 6000; print('OK')"`

**3. 修改 LLMSummarizingCondenser.condense() 支持迭代更新**

修改 `condense()` 方法（当前 lines 77-111）：
- 在 split head/middle/tail 之后，调用 `self._extract_previous_summary(head)` 检测 head 中是否有上次的 LLM summary event（通过 `event.condensed == True` and `event.source == "condenser:llm"` 识别）
- 如果找到 previous_summary，传给 `_summarize(middle, previous_summary=previous_summary)`
- Summary event 的 content 前拼接 `SUMMARY_PREFIX`
- Summary event metadata 改 `"strategy": "llm_state_serialization"` 和 `"iterative": bool`
- 如果是迭代更新，从 head 中移除旧 summary event（防止 summary 堆叠）

新增两个辅助方法：
- `_extract_previous_summary(head: list[Event]) -> str | None`：遍历 head，找 `_is_llm_summary(e)` 为 True 的 event，剥离 SUMMARY_PREFIX 后返回 raw summary 文本
- `_is_llm_summary(event: Event) -> bool`（staticmethod）：检查 `event.condensed and event.source == "condenser:llm"` 且 strategy 匹配 `"llm_summarizing"` 或 `"llm_state_serialization"`（向后兼容旧 summary）

### Phase B: P0-2 — Tool Output Semantic Collapse (`tool_output_pruner.py`)

**5. 添加 tool type 检测 + 语义折叠函数**

在 `src/governance/condenser/tool_output_pruner.py` 中新增：
- `import hashlib, re` 到文件头
- `PruneConfig` 新增 2 个字段：`semantic_collapse: bool = True`, `hash_dedup: bool = True`（默认启用，向后兼容）
- `_TOOL_TYPE_PATTERNS: list[tuple[str, re.Pattern]]`：6 种 tool type 的检测正则，按优先级排列：
  - `"search"`: 匹配 `grep|rg|ripgrep|\d+ match`
  - `"read"`: 匹配 `Read|Content of|cat|File:`
  - `"command"`: 匹配 `^[$>] |exit code|Error:|bash:`
  - `"git"`: 匹配 `^git |diff --git|commit [0-9a-f]|On branch`
  - `"ls"`: 匹配 `^total \d|^d[rwx-]|listing:`
  - `"write"`: 匹配 `wrote|created|saved|Write|Edit`
- `_detect_tool_type(content: str) -> str`：检查前 500 chars，first match wins，无匹配返回 `"unknown"`
- 4 个语义折叠函数（每个接受 `content: str, budget: int` 返回 `str`）：
  - `_collapse_search()`: 保留命令行 + match count + 前 5 + 后 2 条匹配 + summary 行
  - `_collapse_read()`: 保留文件名 + 总行数 + 前 8 行（imports/declarations）+ 后 4 行
  - `_collapse_command()`: 保留命令行 + 全部 error 行（max 10）+ 后 5 行 output + exit code
  - `_collapse_git()`: 保留 branch/status/changed files/commit hashes/diff stats 关键行（max 30 行）
- `_COLLAPSE_FN: dict[str, Callable]`：映射 tool type → collapse 函数。`ls`/`write`/`unknown` 不在此 dict 中，走 positional fallback

→ verify: `python -c "
from src.governance.condenser.tool_output_pruner import _detect_tool_type, _collapse_search
assert _detect_tool_type('grep -r pattern .') == 'search'
assert _detect_tool_type('$ ls -la') == 'command'
assert _detect_tool_type('diff --git a/foo b/foo') == 'git'
assert _detect_tool_type('Read file: test.py') == 'read'
assert _detect_tool_type('hello world nothing special') == 'unknown'
# Semantic collapse: search results
search_output = '50 matches found\n' + '\n'.join(f'file{i}.py:10: match' for i in range(50))
collapsed = _collapse_search(search_output, 2000)
assert '50 matches' in collapsed or 'more matches' in collapsed
assert len(collapsed) < len(search_output)
print('OK')
"`

**6. 重构 prune_text() + 提取 _positional_prune()**

修改 `ToolOutputPruner` 类：
- 将当前 `prune_text()` 的 head+tail 逻辑提取为 `_positional_prune(text: str) -> str`（保持原始 R39 行为）
- 新 `prune_text(text: str, tool_type: str = "unknown") -> str`：
  - `len(text) <= trigger_chars` → 原样返回
  - 计算 budget = `max(trigger_chars, len(text) // 3)`（目标压到原文 ~33%）
  - 如果 `config.semantic_collapse` 且 `tool_type in _COLLAPSE_FN` → 调用对应 collapse 函数
  - collapse 结果有效（非空且比原文短）→ 返回
  - 否则 → fallback 到 `_positional_prune(text)`

→ verify: `python -c "
from src.governance.condenser.tool_output_pruner import ToolOutputPruner, PruneConfig
p = ToolOutputPruner()
# Short text passes through
assert p.prune_text('hello', tool_type='search') == 'hello'
# Long unknown text gets positional prune
long_text = 'x' * 2000
result = p.prune_text(long_text, tool_type='unknown')
assert 'pruned by ToolOutputPruner' in result
assert len(result) < len(long_text)
# Semantic collapse disabled falls back to positional
p2 = ToolOutputPruner(PruneConfig(semantic_collapse=False))
result2 = p2.prune_text('grep result\n' + 'file.py:1: match\n' * 200, tool_type='search')
assert 'pruned by ToolOutputPruner' in result2
print('OK')
"`

- depends on: step 5

**7. 添加 content hash dedup 到 condense()**

修改 `ToolOutputPruner.condense()` 方法：
- 在语义折叠之前，如果 `config.hash_dedup` 为 True，执行 `events = self._dedup_by_hash(events)`
- 新增 `_dedup_by_hash(events: list[Event]) -> list[Event]`：
  - 只对 `source in tool_sources` 的 events 计算 hash
  - 跳过 `len(content) < 200` 的短内容（防误判）
  - MD5[:12] 作为 hash key
  - 从 tail→head 扫描（后出现的更相关），保留最后一次出现，标记早期重复为 dedup
  - 返回去重后的 events 列表
- 语义折叠部分修改 `condense()` 循环：调用 `_detect_tool_type(e.content)` 获取 tool_type，传给 `prune_text(e.content, tool_type=tool_type)`
- Event metadata 新增 `"tool_type": tool_type` 和 `"strategy": "semantic"|"positional"`

→ verify: `python -c "
from src.governance.condenser.tool_output_pruner import ToolOutputPruner
from src.governance.condenser.base import Event, View
# Dedup test: 3 identical tool outputs → keep only the last
dup_content = 'x' * 500
events = [
    Event(id=0, event_type='obs', source='tool', content=dup_content),
    Event(id=1, event_type='obs', source='tool', content='unique content ' * 50),
    Event(id=2, event_type='obs', source='tool', content=dup_content),
    Event(id=3, event_type='obs', source='tool', content=dup_content),
]
p = ToolOutputPruner()
result = p.condense(View(events))
# Should have 2 events after dedup (1 unique + 1 dup kept) + possibly pruned
contents = [e.content for e in result.events]
# The two earlier duplicates (id=0, id=2) should be removed, keeping id=3
assert len(result.events) <= 3  # at most 3 (deduped 2 of 3 identical)
print(f'Dedup: {len(events)} → {len(result.events)} events, OK')
"`
