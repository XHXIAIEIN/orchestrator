## Steps

### Phase 1: Channel-Reducer Protocol (~3h)

1. Create `src/governance/channel_reducer.py` with:
   - `ChannelProtocol(Protocol)` ABC: `update(values: list[Any]) -> bool`, `get() -> Any`, `consume() -> bool`, `finish() -> bool`
   - `LastValueChannel` — stores most recent value, `update()` keeps last item
   - `ReducerChannel(reducer: Callable[[T, T], T])` — applies `reducer(old, new)` on each update (like BinaryOperatorAggregate)
   - `AppendChannel` — `update()` extends list (like Topic), `consume()` clears
   - `MergeChannel(fields: dict[str, ChannelProtocol])` — composite: each field has its own channel type
   → verify: `python -c "from src.governance.channel_reducer import LastValueChannel, ReducerChannel, AppendChannel; import operator; r = ReducerChannel(operator.add, int, 0); r.update([1]); r.update([2]); assert r.get() == 3"`

2. Write `tests/test_channel_reducer.py`:
   - Test LastValueChannel: multiple updates → keeps last
   - Test ReducerChannel with `operator.add`: parallel writes aggregate
   - Test AppendChannel: accumulates, consume clears
   - Test MergeChannel: field-level independent reduction
   - Test edge cases: empty update, get before any update raises EmptyChannelError
   - depends on: step 1
   → verify: `python -m pytest tests/test_channel_reducer.py -v`

   - depends on: step 3

### Phase 3: Interrupt-Resume Mapping (~4h)

5. Add `InterruptPoint` dataclass to `approval.py`:
   ```python
   @dataclass
   class InterruptPoint:
       interrupt_id: str          # xxh3 or sha256 hash of (task_id, node_id, counter)
       task_id: str
       node_id: str               # which agent/step
       counter: int               # nth interrupt in this node
       value: Any                 # payload sent to human
       timestamp: str
   ```
   Add `interrupt_counter: dict[str, int]` to `ApprovalGateway` keyed by `(task_id, node_id)`
   Add `request_interrupt(task_id, node_id, value) -> InterruptPoint` that:
   - Increments counter for (task_id, node_id)
   - Generates interrupt_id from hash of (task_id, node_id, counter)
   - Stores InterruptPoint in `self._interrupts[interrupt_id]`
   - Broadcasts to channels with interrupt_id included
   → verify: `python -c "from src.governance.approval import ApprovalGateway, InterruptPoint; print('import OK')"`

6. Add `resume_interrupt(interrupt_id, value) -> None` to `ApprovalGateway`:
   - Looks up InterruptPoint by interrupt_id
   - Stores resume value in `self._resumes[interrupt_id]`
   - Signals waiting coroutine via asyncio.Event
   - depends on: step 5
   → verify: `python -c "from src.governance.approval import ApprovalGateway; g = ApprovalGateway.__new__(ApprovalGateway); print('resume method exists:', hasattr(g, 'resume_interrupt'))"`

7. Wire interrupt-resume into `executor_session.py`:
   - Add `approval_gateway` parameter to `AgentSessionRunner.__init__()`
   - Add `interrupt(value) -> Any` method that:
     - Calls `gateway.request_interrupt(task_id, current_node, value)`
     - Awaits resume signal
     - Returns resume value to caller
   - This allows mid-session interrupts (not just pre-session approval)
   - depends on: step 6
   → verify: `python -c "from src.governance.executor_session import AgentSessionRunner; print('import OK')"`

### Phase 4: Checkpoint Durability (~2h)

8. Rewrite `checkpoint_recovery.py` to support 3 durability modes:
   - `Durability = Literal["sync", "async", "exit"]`
   - `StructuredCheckpoint` dataclass:
     ```python
     @dataclass
     class StructuredCheckpoint:
         task_id: str
         channel_values: dict[str, Any]     # serialized channel state
         channel_versions: dict[str, int]   # version per channel
         pending_writes: list[tuple[str, Any]]
         metadata: dict
         timestamp: str
     ```
   - `save_checkpoint(cp: StructuredCheckpoint, durability: Durability)`:
     - `sync`: write to events_db immediately, block until done
     - `async`: submit to ThreadPoolExecutor, return immediately
     - `exit`: buffer in memory, flush on task completion or error
   - Keep existing `detect_checkpoint()` as fallback for git-based detection
   → verify: `python -c "from src.governance.checkpoint_recovery import StructuredCheckpoint, Durability; print('OK')"`

9. Wire durability into `executor.py`:
   - Add `durability: Durability = "sync"` parameter to `TaskExecutor.execute_task()`
   - After each successful agent turn, call `save_checkpoint()` based on durability mode
   - On task error/timeout, flush any buffered checkpoints (exit mode safety)
   - depends on: step 8
   → verify: `python -m pytest tests/ -k "executor" -v`

10. Write `tests/test_checkpoint_durability.py`:
    - Test sync mode: checkpoint exists in DB immediately after save
    - Test async mode: checkpoint appears eventually (poll with 1s timeout)
    - Test exit mode: checkpoint NOT in DB mid-task, appears after flush
    - Test crash safety: exit mode + simulated crash → verify last sync checkpoint available
    - depends on: step 9
    → verify: `python -m pytest tests/test_checkpoint_durability.py -v`
